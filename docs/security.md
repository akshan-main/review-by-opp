# Security and Privacy

## Trust model

review-by-opp adds **no new trust requirements** beyond what you already accept by using Claude Code and Codex CLI.

| What | Who sees it |
|------|-------------|
| Your code | Claude (Anthropic) and Codex (OpenAI) - same as using them directly |
| Review findings | Stored locally in `reviews/` - never transmitted |
| Session state | Stored locally in `reviews/` - never transmitted |
| Configuration | Stored locally in `.review-by-opp.json` - never transmitted |

## Local-first design

- **No cloud backend.** The plugin is a set of local files (SKILL.md, hooks, scripts).
- **No API keys required.** Users bring their own signed-in Claude and Codex accounts.
- **No telemetry.** The plugin does not phone home.
- **No external services.** The only network calls are from Claude and Codex themselves, which the user explicitly invokes.

## What data is stored

All in `reviews/` within your project:

- `current.json` - Active session state (findings, config, round history)
- `rounds/round-N.json` - Per-round finding snapshots
- `summaries/summary-N.json` - Per-round summaries
- `artifacts/round-N-*.txt` - Raw review/check artifacts for troubleshooting

This data contains:
- File paths and line numbers from your codebase
- Finding descriptions generated by Codex
- Resolution notes written by Claude

**Recommendation:** Add `reviews/` to `.gitignore` if you don't want review data in version control.

## No UI scraping

The plugin does **not** scrape Claude or Codex chat panes. It uses:
- Codex CLI's documented `exec` command in read-only sandbox mode
- Claude Code's documented plugin skill system
- Standard shell execution for hooks

## Risks

| Risk | Mitigation |
|------|------------|
| Codex sees your code | Same as using Codex directly. Invoked via `codex exec --sandbox read-only`. |
| Claude sees your code | Same as using Claude Code directly. |
| Review data in repo | Add `reviews/` to `.gitignore`. |
| Codex output contains sensitive info | Review data stays local. Don't commit `reviews/`. |
| Shell injection via Codex output | Output is parsed as JSON, not executed as commands. |
